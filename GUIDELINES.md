# Codex Test ガイドライン

## UI概要
- 静的フロントエンド（HTML/CSS/バニラJS）のみで、トライアスロン選手のレース進捗を可視化するビューアを実装しました。
- `data/sample-chip-data.json` のダミーデータを `script.js` で読み込み、選手プルダウンと時間スライダーの操作に応じて区間・進捗バー・残り時間を更新します。
- レイアウトはモバイル縦画面を優先したシングルカラム構成です。

## 実チップデータ投入時の拡張案
- API でのデータ取得：`init` 内の `fetch('data/sample-chip-data.json')` を REST API の URL に差し替え、再描画ロジックを流用する。
- ストリーミング/リアルタイム更新：WebSocket を用いて区間通過イベントを受信し、受信ごとに `athletes` 配列を更新して `updateView` を呼び出す。
- データ型の厳密化：TypeScript などに移行し、スキーマバリデーション（例: JSON Schema）で外部データの信頼性を担保する。
- 大会ごとのデータ切替：クエリパラメータやローカル設定ファイルでロードする JSON を選択できるようにする。
